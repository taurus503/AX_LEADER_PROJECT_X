# 6조 PRD - AI 기반 KOSPI200 옵션 투자전략 검토 및 백테스팅 Agent

## 1. 서비스 개요

### 서비스명

6조 AI 기반 KOSPI200 Option Playbook Advisor

### 한 줄 정의

KOSPI200 시장 스냅샷, 변동성/왜도 점수, 시장 레짐, Option Playbook, 백테스트 결과를 연결하여 옵션 전략 후보와 회피 전략을 한 화면에서 검토하도록 돕는 AI 업무 효율화 Agent입니다.

### 서비스 목표

본 서비스의 목표는 옵션 투자전략 검토 과정에서 반복되는 시장 데이터 확인, 전략 후보 비교, 백테스트 근거 확인, 투자위원회 보고서 초안 작성을 자동화하는 것입니다.

AI가 최종 투자판단을 대신하는 서비스가 아니라, 담당자가 더 빠르고 일관된 기준으로 검토할 수 있도록 시장 상태와 전략 근거를 정리하는 보조 도구입니다.

## 2. 현재 구현 코드 반영 사항

이번 PRD는 기존 PDF/txt 요구서뿐 아니라 다음 두 코드의 실제 구현 내용을 반영합니다.

| 코드 | 반영 내용 |
| :--- | :--- |
| `advisor.js` | `/api/advisor` 요청을 기존 `worker.mjs`의 `fetch()`로 전달하는 API 어댑터입니다. Vercel/Node 환경에서 Cloudflare Worker 방식의 Advisor 로직을 재사용할 수 있게 합니다. |
| `snapshot (1).js` | Yahoo Finance public chart API에서 `^KS200` 데이터를 조회하고, KOSPI200 종가, 20일 실현변동성, 20일 수익률 왜도, 변동성 점수, 왜도 점수, confidence, regime을 계산합니다. |

즉, 현재 MVP는 단순 화면 기획이 아니라 아래 두 축을 이미 갖고 있습니다.

1. 실데이터 기반 KOSPI200 시장 스냅샷 API
2. Option Advisor 화면/Worker 로직을 연결하는 API 라우팅 구조

## 3. 핵심 문제 정의

| 문제 | 설명 |
| :--- | :--- |
| 시장 점검이 수작업 중심 | KOSPI200 지수, 변동성, 뉴스, 리포트, 이벤트 리스크를 각각 확인해야 합니다. |
| 옵션 전략 비교 기준 불일치 | 담당자마다 Covered Call, Cash Secured Put, Long Call, Iron Condor 등 전략의 적합성을 다르게 판단할 수 있습니다. |
| 백테스트 근거 연결 부족 | 전략 추천과 과거 검증 결과가 분리되면 보고서 작성 시간이 늘어납니다. |
| 회피 전략 설명 부족 | 어떤 전략을 피해야 하는지, 왜 위험한지 설명하기 어렵습니다. |
| 투자위원회 보고 부담 | 시장 판단, 전략 추천, 리스크 요약, 검증 근거를 다시 문장화해야 합니다. |

## 4. 사용자 핵심 여정

| 단계 | 사용자 행동 | 시스템 처리 | 산출물 |
| :--- | :--- | :--- | :--- |
| 1. 시장 스냅샷 확인 | 담당자가 기준일의 KOSPI200 상태를 확인합니다. | `snapshot` API가 Yahoo Finance `^KS200` 데이터를 조회합니다. | 종가, 전일 종가, 기준일, 실제 데이터일 |
| 2. 변동성/왜도 확인 | 시장이 고변동인지, 옵션 수요가 어느 쪽으로 기울었는지 봅니다. | 20일 로그수익률 기반 실현변동성과 왜도를 계산합니다. | `realizedVol20`, `skew20` |
| 3. 레짐 분류 확인 | 현재 시장 상태를 한눈에 봅니다. | 변동성 점수와 왜도 점수로 4개 레짐을 분류합니다. | Regime 1-4, confidence |
| 4. 전략 후보 확인 | 담당자가 검토할 옵션 전략을 봅니다. | Advisor가 Option Playbook 기반 Top/Avoid 전략을 제시합니다. | 추천 전략, 회피 전략 |
| 5. 상세 Playbook 확인 | 전략별 손익구조와 리스크를 확인합니다. | 각 전략 상세 페이지로 연결합니다. | 손익 프로파일, Greeks, 백테스트 |
| 6. 투자위원회 보고 준비 | 담당자가 보고서 초안을 작성합니다. | 시장 해석, 추천 사유, 회피 사유, 검증 포인트를 문장화합니다. | Committee Brief 초안 |
| 7. 사후 분석 | 추천 결과가 맞았는지 점검합니다. | 성과 귀인 Agent가 방향성/변동성/시간가치 효과를 분해합니다. | 성과 귀인 리포트 |

## 5. 주요 기능 요구사항

### 5.1 KOSPI200 Snapshot API

현재 `snapshot (1).js` 기준으로 다음 기능이 필요합니다.

- Yahoo Finance public chart API에서 `^KS200` 일간 데이터를 조회합니다.
- 기본 조회 범위는 2년, interval은 1일입니다.
- 사용자가 `date`를 입력하면 해당일 또는 직전 거래일 데이터를 선택합니다.
- 종가, 전일 종가, 실제 데이터 기준일을 반환합니다.
- 20일 로그수익률 기반 실현변동성과 왜도를 계산합니다.
- 최근 rolling data 기준으로 변동성 점수와 왜도 점수를 산출합니다.
- 레짐과 confidence를 반환합니다.
- 응답은 `no-store`로 처리하여 오래된 캐시가 화면에 남지 않게 합니다.

### 5.2 Regime Classification

현재 코드의 레짐 분류는 4개 상태입니다.

| 조건 | 레짐 | 의미 |
| :--- | :--- | :--- |
| `skewScore >= 0`, `volScore >= 0` | Regime 1 | Put skew, High vol |
| `skewScore < 0`, `volScore >= 0` | Regime 2 | Call skew, High vol |
| `skewScore >= 0`, `volScore < 0` | Regime 3 | Put skew, Low vol |
| `skewScore < 0`, `volScore < 0` | Regime 4 | Call skew, Low vol |

향후 PDF 요구서의 10-state 변동성 모델로 확장할 수 있으나, MVP에서는 위 4개 레짐을 우선 사용합니다.

### 5.3 Advisor API

현재 `advisor.js`는 사용자의 `/api/advisor` 요청을 `worker.mjs`로 전달하는 어댑터입니다.

필수 요구사항:

- 요청 method와 header를 Worker의 Fetch API 형식으로 변환합니다.
- Worker 응답의 status, headers, body를 그대로 전달합니다.
- 오류 발생 시 500과 오류 메시지를 JSON으로 반환합니다.
- 화면과 Worker 로직을 분리하여 배포 환경 변경에 유연하게 대응합니다.

### 5.4 Option Playbook 기반 전략 추천

- 48개 KOSPI200 옵션 합성 전략을 전략 저장소로 관리합니다.
- 현재 시장 레짐과 각 전략의 적합성을 비교합니다.
- 추천 전략, 관찰 전략, 회피 전략을 구분합니다.
- 전략별 추천 사유와 회피 사유를 제공합니다.
- 전략 상세 Playbook으로 연결합니다.

### 5.5 백테스트 및 검증

- 2007년부터 2026년 6월까지의 과거 데이터를 기준으로 검토합니다.
- 월간 옵션 만기 사이클을 반영합니다.
- 거래비용 5bps를 고려합니다.
- MDD, VaR, CVaR, Sharpe, Sortino 등 리스크 지표를 제공합니다.
- 2008 금융위기, 2020 코로나 등 스트레스 구간을 별도 점검합니다.

### 5.6 투자위원회 보고서 초안

- 시장 상태 요약
- 추천 전략 및 추천 사유
- 회피 전략 및 회피 사유
- 백테스트 요약
- 리스크 및 추가 검토사항
- 최종 판단은 담당자/위원회 승인 필요 문구

## 6. MVP 범위

### 포함

- KOSPI200 Snapshot API
- 20일 실현변동성 및 왜도 기반 레짐 분류
- Advisor API Worker 연결
- Top Recommended / Avoid Now 전략 표시
- Option Playbook 상세 링크
- 전략별 검토 포인트 및 보고서 초안

### 제외

- AI의 독자적 최종 투자판단
- 실제 주문 자동 제출
- 고빈도 매매
- 확정 수익 보장 표현
- 외부 고객 대상 투자자문
- KOSPI200 옵션 외 자산군 확장

## 7. 성공 기준

| 기준 | 목표 |
| :--- | :--- |
| 업무 시간 단축 | 시장 점검과 전략 비교 시간을 30% 이상 줄입니다. |

| 설명 가능성 | 추천/회피 전략의 사유를 담당자가 설명할 수 있게 합니다. |
| 검증 가능성 | 추천 전략의 백테스트와 리스크 지표를 연결합니다. |
| 운영 안정성 | API 실패 시 오류 메시지와 재시도/검토 필요 상태를 표시합니다. |

## 8. 핵심 메시지

6조의 Option Playbook Advisor는 옵션 전략을 자동으로 매매하는 시스템이 아닙니다. KOSPI200 시장 상태를 실데이터로 점검하고, Option Playbook의 전략 후보를 비교하며, 백테스트와 리스크 근거를 연결해 투자위원회 검토자료를 빠르게 만드는 AX 업무 효율화 도구입니다.
