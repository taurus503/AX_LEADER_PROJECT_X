# design.md — 네오브루탈 화면 규칙

## 기준

삼성생명 민원 지식봇 화면은 친근하지만 가볍지 않은 내부 업무 도구처럼 보여야 한다. 디자인은 TinyFaces 레퍼런스의 강한 섹션 리듬과 카드 구성을 참고하되, NFT/수집품 분위기는 쓰지 않고 민원 응대, FAQ 근거, 담당자 확인 흐름을 우선한다.

## 디자인 토큰

- 폰트: `Pretendard`, `Pretendard Variable`, `Malgun Gothic`, `Arial`, sans-serif 순서.
- 제목: Pretendard 800, letter-spacing 0.
- 본문: Pretendard 계열 400~700, 긴 설명은 line-height 1.55 이상.
- 색상:
  - `--ink: #0a0a0a`
  - `--paper: #f8f5ec`
  - `--blue: #0d63c9`
  - 포인트 노랑: `#ffd84d`
  - 포인트 코랄: `#ff745f`
  - 포인트 민트: `#80e0c0`

## 네오브루탈 규칙

- 카드, 버튼, 입력칸, 히어로 이미지, 상태 카드에는 `3px solid #0a0a0a` 테두리를 사용한다.
- 주요 표면 그림자는 `8px 8px 0 #0a0a0a`로 통일한다. blur, 투명 그림자, 부드러운 elevation은 쓰지 않는다.
- 버튼은 pill 형태를 허용하되 테두리와 hard shadow를 반드시 유지한다.
- 배지는 색상만으로 의미를 전달하지 않고 텍스트를 함께 표시한다.
- `담당자 확인 필요` 배지는 코랄 배경으로 고정한다.
- 과한 애니메이션은 금지한다. 흐르는 텍스트, 무한 슬라이드, 큰 인터랙션 애니메이션은 사용하지 않는다.

## 레이아웃

- 히어로 이미지는 페이지 상단에 독립된 이미지 밴드로 배치한다.
- 히어로 아래에 핵심 문구와 질문 입력 패널을 배치한다.
- 결과와 근거는 각각 카드 또는 접힘 카드로 구성한다.
- 모바일은 항상 1열 구조를 우선한다.
- 데스크톱에서는 문구와 입력 패널을 2열로 둘 수 있지만 읽기 순서는 이미지 → 설명 → 질문 입력 → 결과 → 근거 → 담당자 확인이다.

## 컴포넌트

- 질문 입력칸: 3px 검정 테두리, 8px hard shadow, 명확한 label 제공.
- 실행 버튼: 노랑 배경, 검정 테두리, hard shadow. 비활성 시 shadow 제거 가능.
- 답변 카드: 흰색 또는 아주 연한 민트 배경, 검정 테두리, hard shadow.
- 근거 FAQ: `details/summary` 아코디언을 사용하고, 모바일에서 긴 내용은 접는다.
- 담당자 확인 배지: 코랄 배경, 검정 테두리, 텍스트는 `담당자 확인 필요`.
- 면책/오류 배너: 노랑 또는 코랄 계열 배경, 검정 테두리, 단정하지 않는 문구.

## 접근성

- 본문 텍스트 대비는 4.5:1 이상을 목표로 한다.
- 키보드 포커스는 노랑 outline으로 분명히 표시한다.
- 입력 오류는 `aria-live`로 알린다.
- 모바일 터치 대상은 최소 44px 이상으로 둔다.
- 색상만으로 상태를 구분하지 않고 배지 텍스트와 안내 문구를 함께 제공한다.
