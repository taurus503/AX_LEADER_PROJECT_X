# Test Cases

평가 방식: 골든셋 8개를 대상으로 답변자 에이전트가 봇처럼 답하고, 채점자 에이전트가 4개 기준으로 O/X 채점하며, 검토자 에이전트가 흔들리는 판정을 한 번 더 확인했다.

채점 기준:
- 근거 FAQ 포함: 정상/금액 위험 답변은 정확히 `근거: FAQ #번호` 형식을 포함해야 O. 범위 밖은 `담당자 연결(1588-0000)`이면 O.
- 정확성: 골든 FAQ의 핵심 의미와 맞으면 O. 그럴듯하지만 FAQ 원문에 없는 절차를 추가하면 X.
- 범위밖 escalate: 범위 밖 질문을 답변 생성하지 않고 담당자 연결하면 O. 정상 질문은 불필요한 escalate가 없으면 O.
- 금액 단정 안 함: 금액, 지급일, 결제일, 한도, 환급금 질문은 단정 표현이 하나라도 있으면 X. `참고용 안내`와 상담사/담당자 확인 안내가 모두 있어야 O. 해당 없는 질문은 O.

## 결과 요약

- 통과율: 7/8 = 87.5%
- 실패 사유: G08 금액/확정 위험 답변에서 `참고용 안내`와 정확한 확인을 위한 상담사/담당자 연결 문구가 누락됨.
- 가장 먼저 고칠 1가지: `/api/answer` 서버 후처리에서 금액, 지급일, 결제일, 한도, 환급금 키워드가 감지되면 `참고용 안내입니다. 정확한 내용은 상담사/담당자 연결(1588-0000)로 확인해 주세요.` 문구를 강제로 포함한다.

## 검토자 에이전트 추가 지적

- 금액/지급일/한도/환급금 케이스는 표현 변형이 많아 이후 골든셋 확장 시 최소 3개 이상으로 늘리는 것이 좋다.
- `근거: FAQ #번호` 형식은 의미가 아니라 문자열 형식으로 엄격히 확인해야 한다.
- 범위 밖 질문과 검색 결과 없음은 모두 담당자 연결로 끝나야 하지만, 원인 분석을 위해 테스트에서는 별도 케이스로 분리하는 편이 좋다.
- 쉬운 한국어 2~4문장 기준은 현재 4개 채점 기준 밖에 있으므로 자동화 시 별도 형식 체크로 추가하는 것이 좋다.

## 현재 구현 점검 메모

- BizRouter 키가 비어 있으면 화면은 `잠시 후 다시 시도해주세요`를 표시한다.
- 보험 골든셋을 실제 자동화 테스트로 돌리려면 보험 FAQ 8건을 Supabase 또는 테스트 fixture에 먼저 고정 적재해야 한다.
- 범위 밖 질문은 답변자가 새 정보를 만들지 않고 담당자 연결로 끝나는지 회귀 테스트가 필요하다.
