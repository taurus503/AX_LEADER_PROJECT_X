# Agent 기획서 (PRD) — ○○생명 민원 지식봇  〔D4_03 기획서 1페이지 양식〕

> D3 As-Is·To-Be → D4 기획서 → D5 설계 → D6 완성으로 이어지는 시작 문서(베스트 프랙티스). 팀 맥락에 맞게 한 줄씩 수정. 공개·가상·더미(Public-First).

| 칸 | 내용 |
|---|---|
| **문제 정의** | (D3 근거 숫자 + 대상 + 현재 불편) 보험금 청구의 '서류보완요청→재접수' 구간이 반복·지연(보완 33%·p90 큼). 상담사는 반복 검색·안내, 고객은 재제출 대기. |
| **대상 사용자 / JTBD** | 민원·CS 상담사/기획 담당자. "반복 민원에 근거 있는 초안 답변을 빠르게 만들고, 위험한 답은 사람에게 넘기고 싶다." |
| **핵심 기능 1** | 질문 → 관련 FAQ 검색 → 근거(FAQ 번호) 포함 답변 초안 (입력 질문 / 처리 검색+LLM / 출력 답변+근거) |
| **핵심 기능 2** | 질문 자동 분류 → 해당 카테고리 FAQ만 검색(라우팅) (입력 질문 / 처리 분류 / 출력 카테고리) |
| **안 만들 것(보류·금지)** | 실고객·계약 데이터 연동(보안), 보험금/해지 확정 판단(책임), 자동 발송(되돌릴 수 없음) |
| **입력 데이터** | 공개·더미 FAQ(category·question·answer·source). 실데이터는 AI Hub(71844) 교체. |
| **출력 형식** | 화면(웹) + 답변 JSON(answer·sources·escalate) |
| **사람 승인 지점(HITL)** | 고객 발송 전 직원 확인 / 금액·확정·범위 밖 → 상담사 연결 |
| **완료 정의** | 더미 5문항 중 4개가 근거(FAQ #) 포함 답변, 범위 밖 1문항은 상담사 연결(escalate=true) |
| **D5 인계 메모** | 입력=질문, 처리=분류→검색→LLM, 출력=답변+근거+상담사연결 |
| **보류 이유(한 줄)** | 실데이터·확정판단·발송은 보안·책임·가역성 때문에 2주차 후반·현업 검토 뒤 |

## 후보 검증 질문 5개
1. 가장 약한 가정은? (검색만으로 근거 답변이 충분한가)
2. 데이터가 부족한 부분은? (FAQ 커버리지)
3. 보안 위험은? (개인정보·키 노출)
4. 비용 위험은? (LLM 호출 횟수·모델)
5. 사람 승인 지점은 충분한가? (금액·발송·범위 밖)
