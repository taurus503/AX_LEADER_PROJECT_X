# 1분 발표 텍스트 - 삼성생명 민원 지식봇

## 1장. AS-IS 문제

현재 보험금 청구와 민원 상담에서는 고객 질문마다 담당자가 FAQ를 다시 찾고, 서류 보완 여부나 답변 초안을 반복해서 작성합니다. 특히 서류보완 요청, 재접수, 진행상태 문의가 반복되면 고객은 기다리고 담당자는 검색과 확인에 시간을 씁니다. 더 어려운 점은 금액, 지급 여부, 보장 범위처럼 확정적으로 말하면 안 되는 질문도 섞여 있다는 점입니다.

## 2장. 데이터 근거

PRD 기준으로 입력 데이터는 category, question, answer, source 형태의 FAQ입니다. 목표는 샘플 질문 5개 중 4개 이상에서 FAQ 번호가 포함된 근거 답변을 만들고, 범위 밖 질문 1개는 담당자 연결로 보내는 것입니다. 즉 이 PoC는 AI가 판단을 대신하는 것이 아니라, 반복 질문에서 근거 있는 초안을 만들 수 있는지 검증하는 단계입니다.

## 3장. TO-BE(이 봇)

TO-BE 화면에서는 고객 질문을 입력하면 먼저 카테고리를 예측하고, 해당 카테고리의 FAQ를 우선 검색합니다. 이후 참고한 FAQ 카드와 함께 BizRouter를 통해 쉬운 말 답변 초안을 만들고, 마지막에는 근거 FAQ 번호를 붙입니다. 금액, 결제일, 한도, 환급금처럼 위험한 질문은 참고용 안내와 담당자 확인으로 남겨 고객 발송 전 사람이 최종 확인합니다.

## 4장. 기대효과(가설)

가설은 세 가지입니다. 첫째, 담당자가 반복 FAQ를 찾는 시간을 줄일 수 있습니다. 둘째, 답변마다 근거 FAQ가 남아 품질 점검과 교육에 활용할 수 있습니다. 셋째, 위험한 확정 답변을 자동으로 걸러 담당자 확인으로 넘기면 민원 리스크를 낮출 수 있습니다. 이번 PoC는 작은 FAQ부터 시작해, 실제 상담 데이터와 권한 통제가 준비되면 업무 적용 가능성을 검증하는 출발점입니다.
