# External Data Review Design

## Goal

정적 투자규제 검토 페이지를 유지하면서, 검토 결과를 서버 함수에서 생성하고 공식 법령 원문 링크를 함께 내려주는 구조로 전환한다.

## Chosen Approach

1. 프론트는 기존 입력 화면과 결과 탭 구조를 유지한다.
2. `POST /api/review` 서버 함수가 검토 로직을 수행한다.
3. 법조문은 공식 원문 기준으로 정리한 소스 팩을 사용하고, 각 요청 시 국가법령정보센터 링크 연결 상태를 함께 확인한다.
4. 공시/보고처는 조문 문언에서 직접 확인되는 범위만 표기하고, 직접 확인되지 않으면 `근거없음`으로 남긴다.

## Why

- 브라우저에 키나 내부 판단 로직을 노출하지 않는다.
- 배포 구조를 크게 바꾸지 않고 Vercel 함수만 추가하면 된다.
- 사용자가 원한 "투자자 관점 + 법조문 근거 + 한눈에 보이는 출력"을 유지할 수 있다.

## Limits

- 법령 원문 전체를 실시간 API로 직접 당겨오는 구조는 아니다.
- 공식 링크 확인은 실시간이지만, 조문 텍스트 팩은 구현 시점의 공식 원문 기준으로 묶였다.
- 조문에 공시 위치가 직접 적혀 있지 않은 경우 `근거없음` 처리한다.
