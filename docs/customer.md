# 6조 고객 정의서 - AI 기반 KOSPI200 Option Playbook Advisor

## 1. 타겟 사용자

본 서비스의 사용자는 KOSPI200 옵션 전략을 직접 운용하거나, 전략 검토자료를 작성하거나, 리스크 관점에서 승인 가능성을 확인하는 내부 사용자입니다.

| 사용자 유형 | 설명 |
| :--- | :--- |
| 투자전략 실무자 | 시장 상태와 옵션 전략 후보를 매일 점검하는 담당자 |
| 자산운용/ALM 담당자 | 시장 국면에 맞는 전략 후보와 위험 요인을 검토하는 사용자 |
| 리스크관리/재무심사 담당자 | 손실 가능성, 검증 근거, 통제 장치를 확인하는 검토자 |
| 투자위원회 보고 담당자 | 전략 추천 사유와 백테스트 근거를 보고서로 정리하는 담당자 |
| 팀장/의사결정권자 | AI 산출물이 승인 가능한 근거를 갖췄는지 판단하는 관리자 |
| DX Leaders | 반복 업무를 AI Agent 기반 업무 흐름으로 개선하려는 내부 혁신 담당자 |

## 3. 사용자가 겪는 주요 고충

### 3.1 시장 데이터 확인이 번거로움

KOSPI200 지수, 전일 대비 변화, 변동성, 이벤트 리스크를 각각 확인해야 합니다. 현재 `snapshot` API는 Yahoo Finance `^KS200` 데이터를 직접 조회하므로 이 문제를 줄일 수 있습니다.

### 3.2 옵션 전략 용어가 어렵고 비교가 복잡함

Covered Call, Cash Secured Put, Long Call, Iron Condor, Long Straddle, Long Strangle 등은 손익구조와 적합 국면이 다릅니다. 사용자는 각 전략이 언제 유리하고 언제 위험한지 빠르게 알고 싶어합니다.

### 3.3 추천 결과의 근거가 부족하면 신뢰하기 어려움

AI가 "추천"만 제시하면 팀장이나 심사 담당자는 받아들이기 어렵습니다. 기준일, 데이터 출처, 변동성 점수, 왜도 점수, confidence, 백테스트 결과가 함께 필요합니다.

### 3.4 백테스트와 현재 시장 판단이 분리됨

추천 전략이 과거 유사 국면에서 어떤 성과를 냈는지 바로 확인하지 못하면 투자위원회 자료로 쓰기 어렵습니다.

### 3.5 API나 데이터 오류가 발생했을 때 판단이 어려움

Yahoo Finance 응답 실패, 데이터 부족, 기준일 미일치, Worker 호출 실패가 발생할 수 있습니다. 사용자는 오류 상황에서 AI가 억지로 추천하지 않고 "검토 필요"로 표시하기를 기대합니다.

## 4. 서비스 사용 이점

| 이점 | 설명 |
| :--- | :--- |
| 실데이터 기반 시장 점검 | `^KS200` 일간 데이터를 사용해 기준일의 시장 상태를 확인합니다. |
| 레짐 판단 표준화 | 20일 실현변동성과 왜도 점수를 기준으로 Regime 1-4를 분류합니다. |
| 전략 비교 시간 단축 | Option Playbook 기반으로 추천/회피 전략을 한 화면에서 비교합니다. |
| 보고서 작성 효율화 | 시장 해석과 전략 근거를 투자위원회 보고서 초안으로 전환합니다. |
| 리스크 검토 강화 | 백테스트, 스트레스 테스트, MDD, VaR/CVaR 연결을 통해 검증 가능성을 높입니다. |
| 운영 안정성 확보 | API 오류, 데이터 부족, 낮은 confidence 상황을 별도 표시할 수 있습니다. |

## 5. 사용자별 핵심 요구

| 사용자 | 핵심 요구 | 반영 기능 |
| :--- | :--- | :--- |
| 투자전략 실무자 | 오늘 어떤 전략을 먼저 볼지 알고 싶음 | Top Recommended, Regime, confidence |
| 리스크 검토자 | 왜 위험한지 설명 가능해야 함 | Avoid Now, MDD, VaR/CVaR, stress test |
| 보고 담당자 | 보고서 초안이 필요함 | Committee Brief, 추천/회피 사유 자동 문장화 |
| 팀장 | AI 판단의 근거와 한계를 보고 싶음 | 기준일, 출처, 오류, 사람 승인 문구 |
| DX Leaders | 업무 흐름 개선 효과가 필요함 | Before/After, API 자동화, 표준 프로세스 |

## 6. 고객 관점 핵심 메시지

이 서비스는 옵션 전문가만을 위한 도구가 아닙니다. 옵션 전략을 잘 모르는 사용자도 현재 시장 상태, 추천 전략, 회피 전략, 검증 근거를 같은 기준으로 이해할 수 있게 해주는 내부 업무 보조 도구입니다.

특히 6조 서비스의 강점은 실제 코드상으로 KOSPI200 시장 스냅샷 API와 Advisor API 연결 구조가 이미 반영되어 있다는 점입니다. 따라서 고객에게는 "아이디어"가 아니라 "실제 데이터로 동작하는 MVP"로 설명할 수 있습니다.
